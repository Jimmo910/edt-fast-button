# Security Policy

## Reporting a vulnerability

Do not disclose a suspected vulnerability in a public issue. Use GitHub's **Security > Report a vulnerability** form
for this repository. If private vulnerability reporting is unavailable, contact the maintainer through the private
contact information on the `Jimmo910` GitHub profile.

Include the affected version, EDT version, reproduction steps, expected impact, and any suggested mitigation. Avoid
including real credentials, private repository URLs, or other sensitive data. You should receive an acknowledgement
within seven days; release timing will depend on severity and the availability of a safe fix.

Only the latest released version is supported with security fixes.
