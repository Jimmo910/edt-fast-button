# Журнал изменений

Здесь фиксируются все значимые изменения проекта. Формат соответствует Keep a Changelog, а версии — Semantic
Versioning.

## [Unreleased]

## [0.1.5] - 2026-07-15

### Изменено

- Сборка переведена на платформу EDT 2026.1. Плагин по-прежнему рассчитан на Java 17, а диапазоны зависимостей уже покрывают EGit/JGit 6.8.12.

## [0.1.4] - 2026-07-15

### Добавлено

- Кнопка **«Слить…»** для Git-проекта, использующая штатное окно слияния и иконку EGit.
- Переключатели видимости для каждой кнопки в разделе **Окно > Параметры > Быстрые кнопки** — любое действие можно
  показать или скрыть (по умолчанию показаны все).

## [0.1.3] - 2026-07-14

### Добавлено

- В верхнюю часть контекстного меню Git-проекта добавлена кнопка «Переключиться на другую ветку...» со штатной
  Git-иконкой и окном EGit для выбора произвольной ветки, тега или ref.

### Изменено

- Расширены диапазоны зависимостей JGit и EGit до `[6.8.0,8.0.0)`, чтобы плагин устанавливался и на следующей мажорной версии EDT.

## [0.1.2] - 2026-07-14

### Исправлено

- Апдейт-сайт больше не показывает плагин трижды в окне «Установить новое ПО»: теперь предлагается одна фича в одной категории.

### Изменено

- Документация теперь в первую очередь на русском: README.md стал русским, английская версия перенесена в README_EN.md.
- Заметки к релизам на GitHub публикуются на русском из CHANGELOG_RU.md.

## [0.1.1] - 2026-07-14

### Изменено

- Поиск репозиториев и сопоставление проектов теперь выполняются в фоновой задаче, а не в потоке интерфейса, что
  сохраняет отзывчивость рабочей области на больших воркспейсах.
- Оркестрация фоновой задачи теперь покрыта headless-модульными тестами через интерфейс резолвера репозиториев и
  фабрику сценариев использования.
- Сообщение об успехе теперь также появляется в строке состояния при активном редакторе, а не только для представлений.
- Локализованное форматирование сообщений централизовано в одном протестированном резолвере; его покрытие теперь
  измеряется и контролируется.
- Анализ SonarQube Cloud пропускается для pull request из форков, чьи запуски не имеют доступа к токену Sonar, —
  внешние вклады больше не блокируются обязательной проверкой.
- Шаги GitHub Actions закреплены по хэшам коммитов.
- SonarQube Cloud теперь требует реальный токен, ожидает Quality Gate и классифицирует Tycho-бандл с тестами как
  тестовый код для точных проблем и покрытия.
- Названия проверок CI и ограничения Dependabot для Java 17 указаны явно и подходят для защиты веток.
- Жизненный цикл Eclipse, владение репозиторием, результаты обработчиков и обход несохранённых редакторов уточнены
  без изменения видимого пользователю поведения.

## [0.1.0] - 2026-07-13

### Добавлено

- Настройка рабочей области для целевой ветки, локализованная на английском и русском языках.
- Команда навигатора EDT, работающая только с Git, размещённая перед стандартной группой New.
- Предусловия по чистому дереву и несохранённым редакторам.
- Получение изменений из upstream, безопасное переключение, создание локальной ветки и обновления только через
  fast-forward.
- Явная обработка отсутствующих remote, отсутствующих веток, опережения и расхождения историй, небезопасного
  состояния репозитория, отмены и обновления проекта.
- Интеграционные тесты JUnit, Checkstyle, порог покрытия JaCoCo, CI, CodeQL, интеграция с SonarQube Cloud и
  автоматические аттестованные архивы релизов p2.
